## עץ 8 — Dreaming (ADHD-like link creation)

### הרעיון
מוח של אדם עם ADHD עובר mind-wandering: הסף להפעלת קשר אסוציאטיבי נמוך
מהרגיל, ולכן יותר "קפיצות רעיוניות" מתרחשות. כשזה מכוון היטב, הצפייה
הרחבה הזו יכולה לפרנס hyperfocus על רעיון אחד טוב.

### היישום
```rust
pub fn propose_via_two_hop(
    store: &AtomStore,
    seeds: &[AtomId],
    max_per_seed: usize,
    proposal_seed: u64,       // deterministic ordering
) -> Vec<CandidateEdge>
```
לכל seed A: אם A→X→B קיים אבל A→B לא, מציעים A→B כ-candidate.
הסדר של ה-candidates נקבע על ידי hash deterministic — אותו proposal_seed
תמיד יחזיר אותם הצעות באותו סדר.

### הוכחה בדמו
ב-`live-brain-demo` conversation 2 (יצירתית), המערכת הציעה:
- `service_container → Eloquent_ORM` — התקבל
- `service_container → PHP` — התקבל

אלה קשרים חדשים שלא היו בגרף קודם. המערכת **ממציאה ידע חדש** על בסיס
מבני קיים, ובודקת אותו דרך evaluation.

## עץ 9 — 3-Stage Evaluation Loop

### הרעיון
לא כל candidate טוב. צריך מנגנון שמחליט "זה קשר אמיתי?" או "רק רעש?"
לפי 3 קריטריונים:

1. **Local strength** — האם A ו-B כבר חולקים context (common neighbors)?
2. **Provenance integrity** — האם hash של המקור תקין? (hook לעתיד)
3. **Global consistency** — האם הקשר החדש סותר קשרים קיימים?

### היישום
```rust
pub fn evaluate(
    store: &AtomStore,
    candidate: &CandidateEdge,
    acceptance_threshold: f32,
) -> EvaluationResult {
    let local_strength = shared_neighbors / max_neighbors;
    let provenance_ok = true;  // hook
    let consistency_ok = !duplicate && !contradicts;
    EvaluationResult {
        accepted: local_strength >= threshold && provenance_ok && consistency_ok,
        ...
    }
}
```

### Recursive hyperfocus
אם candidate עבר, הצד היעד הופך ל-seed חדש ל-next iteration.
ADHD mind-wandering + hyperfocus feed each other:
- רחב (low threshold) → candidate proposals
- עמוק (accepted candidates become new seeds) → hyperfocus

## עץ 10 — Skills grow with need

### הרעיון
כל כישור (תכנותי, רגולטורי, קוגניטיבי) הוא atom בגרף. כשמשתמש משתמש
בכישור בהצלחה, המשקל שלו עולה. כשלון — יורד. אחרי זמן, הגרף יודע
**מה האדם יודע לעשות** ובאיזה רמה.

### היישום
```rust
register_skill(store, "laravel_mvc")         // content-hash dedup
  → AtomKind::Concept with data = "skill:laravel_mvc"
attach_skill(store, person, skill, weight)   // has_skill edge
reinforce_skill(store, person, skill, success, timestamp)
  success: weight += 5 (max 100)
  failure: weight -= 3 (min 0)
  + adds audit trail atom
skill_solves(skill, problem)    // used_for edge
skill_requires(skill, prereq)   // requires edge
```

### Proficiency buckets
- Novice (0-30)
- Developing (31-60)
- Proficient (61-85)
- Mastered (86-100)

### Queries מוכיחות
```rust
skills_of(person)              → Vec<(skill_id, weight)> sorted desc
people_with_skill(s, min_prof) → Vec<(person, weight)>
skills_for_problem(problem)    → Vec<skill_id>
```
כל המילטרים עובדים על פי content-hash + weight max per edge.

## עץ 11 — Dirichlet Meta-Learning

### הרעיון
איזה cognitive mode (Precision/Divergent/Gestalt/Narrative) הכי מתאים
לאיזה סוג שאילתה? זה לא נקבע מראש — המערכת **לומדת** מאינטראקציות.

Dirichlet distribution = conjugate prior של Multinomial. פרמטרים α₁..α₄
מתחילים ב-[1,1,1,1] (uniform). כל הצלחה של mode i → αᵢ += usefulness.
Posterior mean = αᵢ / Σα = סיכוי נורמלי שנבחר mode i.

### היישום
```rust
ModeWeights {
    alpha: [f32; 4],
    total_observations: u64,
}
    record(mode, usefulness)        // Bayesian update
    posterior_mean() -> [f32; 4]
    best_mode()      -> argmax
    sample_by_hash(query_hash) -> deterministic proportional sample
    entropy() / confidence()
```

### MetaLearner — context-specific learning
```rust
MetaLearner {
    per_context: HashMap<String, ModeWeights>,  // "factual", "creative", ...
    global: ModeWeights,
}
```
שאילתות "factual" לומדות בנפרד מ-"creative". Fallback ל-global לcontext
חדש שלא ראינו.

### מבחן אמיתי
Test `sample_by_hash_proportional_across_many_queries`:
- 90 הצלחות Precision, 10 הצלחות Divergent
- Posterior: Precision ≈ 0.89, Divergent ≈ 0.10
- 10,000 queries → >80% רוטטו ל-Precision (דטרמיניסטי לפי hash)

## Scenario — איחוד של שיחות כ-subgraphs

תוספת חשובה שהייתה בשיחה של עידן (ולא אחת מה-11 עצים): כל שיחה צריכה
להיות subgraph קבוע בגרף הפרסונלי. זה מה ש-`scenario.rs` עושה:

```rust
ScenarioBuilder::new(store, person, timestamp, label)
    .mentioned_all(&[laravel, mvc, controller])
    .with_emotion(joy)
    .commit()
```

יוצר atom מסוג Relation עם data = "scenario:TIMESTAMP:label", מחובר
ל-person (agent_of), למונחים (co_occurs_with), ולרגש (emotion_triggered).

### הפונקציה שפותרת "לפני שנתיים":
```rust
reactivate_by_similarity(store, person, current_atoms, now, tau, sim_weight)
```
Jaccard overlap של current_atoms עם scenario mentions + temporal decay.
עם `similarity_weight = 0.9`, scenario דנטלי מלפני 10*tau (=שנה)
יעבר scenario פיצה מאתמול כש-current = {tooth, dentist}.

**זה מוכיח את מה שעידן שאל:** "לפעמים זה דקות ולפעמים שנים" — נפתר
על ידי שילוב של recency (temporal decay) עם similarity (topic match).

## מה שחסר עדיין

### עץ 2 — Selective encryption ברמת עמודה
יש 6 scopes (system/lang/data/family/org/personal) ויש AES-256-GCM ברמת WAL.
אין per-column encryption לעמודות רגישות (PII) באתומים עצמם. זה דורש:
- סימון field של atom כ-"sensitive"
- wrapping ב-AEAD (AES-GCM או XChaCha20-Poly1305) עם מפתח per-scope
- ספירות per-IV

### חיבורים חסרים בין modules
1. **Hopfield + AtomStore**: כרגע HopfieldBank נפרד. צריך atom_id → vector mapping
   שב-AtomStore שומר embeddings optionally. זה יאפשר spread activation על
   מצע embeddings + hopfield atoms ביחד.

2. **Meta-learner + cognitive_modes**: `MetaLearner` יודע לבחור mode, אבל
   אף פונקציה של מצב הליכה לא "שואלת" אותו. צריך wrapper `smart_walk(query)`
   שמשתמש ב-meta לבחור mode, ומעדכן אחרי תוצאה.

3. **Dreaming + Session**: `dream()` מקבל seeds. צריך לחבר אותו ל-SessionContext
   כך שהmind-wandering ימשיך מהאטומים הפעילים עכשיו.

4. **Scenario + Session**: כשסשן נגמר (pruning → empty), הפעלת ScenarioBuilder
   אוטומטית עם הaktoms שהיו בסשן. אוטומציית "שמירת זיכרון" של השיחה.

## Deterministic guarantee

כל ה-code שנוסף בסשן הזה עומד בכלל הזהב:
- `propose_via_two_hop` משתמש ב-`pseudo_hash(from, to, seed)` — לא rand
- `sample_by_hash` משתמש ב-`query_hash` → bucket → CDF — לא rand
- `temporal_weight(created, now, tau)` — formula מתמטית
- `reinforce_skill` — +5/-3 קבועים

כל הטסטים כוללים `determinism_same_*_same_*` שמוודא שני runs זהים נותנים
זהה state.
