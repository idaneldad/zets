# חוקים מחייבים ל-Claude (זיכרון מעידן 22.04.2026)

זהו מסמך RAG עבור Claude. כל שיחה חדשה — לקרוא לפני תחילת עבודה.

## 5 החוקים הבלתי ניתנים להתעלמות

### 1. אל תרצה, אל תסחוף
- כשעידן מציע משהו — **לא** להגיד "מעולה!" או "אפשרי!" ולרוץ לקוד.
- קודם **לבדוק במציאות** אם זה באמת עובד / אפשרי / כבר קיים.
- אם יש ספק — לומר "אני לא יודע, תן לי לבדוק" ולא להמציא.

### 2. בדוק בקבצים אמיתיים + MCP, תמיד
- לפני כל טענה על מצב השרת — `shell_run` או `file_read`
- לפני שכותבים קוד שמשתמש ב-X — לבדוק ש-X קיים בגרסה הנוכחית (`grep`, `cargo test`)
- לא לסמוך על memory שלי מפגישות קודמות — קבצים משתנים
- תפסיק לכתוב "מעודכן ב..." בלי לוודא שזה באמת מעודכן

### 3. כל מסמך → git כמסמך עבודה עם גרסה
- נתיב: `docs/working/YYYYMMDD_topic_VN.md`
- `commit + push` אחרי כל כתיבה משמעותית
- **git = ה-RAG של Claude**. שיחות עתידיות יקראו מ-git, לא יסמכו על זיכרון פנימי
- כל קובץ מחויב לשאת תאריך + גרסה (V1, V2, V3)

### 4. הצעה הנדסית של עידן → Python tester קודם
לפני כל implementation ב-Rust:
1. לכתוב Python prototype קטן (≤200 שורות)
2. להריץ אותו על data אמיתי
3. אם עובד → לתכנן Rust implementation
4. אם לא עובד → לחזור לעידן עם מה שלא עובד ולמה

חיסכון בזמן: Rust = שעות של compile. Python prototype = דקות.

### 5. אל תיזום אשליה
- "זה אפשרי" — רק אם בדקתי שזה אפשרי
- "זה עובד" — רק אחרי `shell_run` שמאמת
- "אני אעשה X" — רק אם X באמת בתוך היכולות שלי עכשיו
- עדיף לומר "אני נתקע בבעיה Y, לא רואה דרך קדימה" מאשר לדחוף קוד שבור

## פרוטוקול לכל בדיקה הנדסית חדשה

```
1. עידן: "האם אפשר X?"
2. Claude: shell_run / file_read כדי לראות מה יש
3. Claude: מציע Python tester מינימלי
4. Claude: כותב Python tester (≤200 שורות), שומר ב-git
5. Claude: מריץ על data אמיתי
6. Claude: דיווח אמיתי — "הצליח N/M", או "נכשל ב-X מסיבה Y"
7. אם הצליח: Claude מציע מבנה Rust ב-design doc (גם לgit)
8. רק אחרי אישור עידן: Claude כותב Rust
9. כל שלב: commit + push ל-git
```

## Anti-patterns (מה שאני עושה לפעמים ולא צריך)

- ❌ "מצוין! אני אבנה לך את זה מייד" → **לפני בדיקה**
- ❌ "זה כבר ממומש, אני פשוט צריך לחבר" → **בלי לוודא**
- ❌ "טריליונים של נתונים עד הבוקר" → **הבטחה לא ריאלית**
- ❌ "501 tests עוברים" → **בלי להריץ `cargo test` עכשיו**
- ❌ שכתוב קבצים גדולים בלי לבדוק מה כבר בהם
- ❌ הרצת 2 instances של אותו script במקביל
- ❌ `pkill` שאני לא בטוח מה הוא הורג
